# كثّرها (Kathirha) — هاكاثون امد ٢٠٢٦

**لعبة ادّخار عادلة داخل تطبيق الإنماء — تنافس بنقاط تكسبها بجهدك، لا بحجم ثروتك.**

مشروع فردي: **أنس السبيعي** — الفكرة والتنفيذ كاملًا (Backend، واجهة عربية، ذكاء اصطناعي، تكامل مصرفي).

---

## الفكرة في سطرين

الادّخار مملّ وبلا حافز، وأدوات اليوم تكافئ من يملك أكثر — لا من يجتهد أكثر.
«كثّرها» تحوّل الادّخار إلى دوريات إنجاز عادلة: **النقاط تُكتسب ولا تُنقَص عند الصرف**، والادّخار يمنح نقاطًا **حتى 40% من دخلك** — فالطالب الذي يدّخر 30% من راتبه الصغير يتفوّق على من يدّخر مبلغًا أكبر بنسبة أقل.

## أبرز ما هو مبني ويعمل

- **محرّك التصنيف العادل** — الترتيب بالنقاط المكتسبة (لا تُنقَص عند الصرف)؛ الادّخار يمنحها حتى 40% من الدخل، والباقي من المهام وسؤال اليوم والاستبيانات. الدوريات دوريات إنجاز (برونزي ← ماسي)، لا تعتمد على الراتب أبدًا.
- **منع التلاعب** — المنافسة للحسابات الموثّقة بنكيًا فقط، والإيداع الذي يُسحب سريعًا لا يُحتسب ادّخارًا.
- **مدرّب ادّخار ذكي** — شخصية الإنفاق، مهام من إنفاقك الفعلي، سؤال مالي يومي، خطط أهداف، وإنقاذ الهدف المتأخر عبر WhatsApp.
- **دقّة مُقاسة** — مخرجات الذكاء الاصطناعي فُحصت آليًا بالكود: نجاح 50 من 50 في الاختبارات الأساسية، وكل النقاط تمنحها الشيفرة لا النموذج (توافق شرعي بالتصميم).
- **بطاقات كثّرها** — 13 تصميم بطاقة قابلة للاقتناء: بطاقات أبطال تُمنح بالمراكز والدوريات ولا تُشترى أبدًا، وتصاميم تُفتح بنقاط المحفظة، وبطاقة موسمية حصرية.
- **وضع عرض التحكيم** — زر «عرض تجريبي فوري» في صفحة الدخول يبني سيناريو كاملًا بنقرة واحدة.

## التشغيل (بدون أي إعدادات خارجية)

يعمل النظام بقاعدة بيانات H2 داخل الذاكرة وببيانات تجريبية تُزرع تلقائيًا — لا يحتاج أي مفاتيح أو تثبيتات إضافية.

**التطبيق كاملًا (واجهة + Backend):**

```bash
# الطرف الخلفي — على المنفذ الذي تتوقعه الواجهة
SERVER_PORT=8099 mvn -f backend/pom.xml spring-boot:run
# (PowerShell:  $env:SERVER_PORT='8099'; mvn -f backend/pom.xml spring-boot:run)

# الواجهة — في طرفية ثانية
cd frontend && npm install && npm run dev
# افتح http://localhost:5173 واضغط «عرض تجريبي فوري (بنقرة واحدة)»
```

**الـ API وحده:**

```bash
cd backend
mvn spring-boot:run
# API on http://localhost:8080
```

ثم لتجهيز سيناريو تحكيم كامل بنداء واحد:

```bash
curl -X POST http://localhost:8080/api/demo/seed
# يرجع token جاهزًا + مستخدم موثّق بمعاملات ودخل مكتشف ومهام وهدف ونقاط
```

جرّب بعدها مثلًا:

```
GET  /api/dashboard          → لوحة المستخدم الكاملة (Bearer token)
GET  /api/leaderboard        → الترتيب العادل بالنقاط
GET  /api/questions/today    → سؤال اليوم
POST /api/questions/answer   → أجب واكسب نقاطًا فورًا
```

المرجع الكامل: [`docs/API.md`](docs/API.md)

**اختبارات:** `mvn test` — تشمل اختبار رحلة التحكيم كاملة من البذر حتى النقاط.

> ملاحظة: الذكاء الاصطناعي يعمل بمحرّك قواعد مدمج دائم الجاهزية؛ وعند توفير `OPENAI_API_KEY` تتحسّن الصياغة تلقائيًا دون أي تغيير في المنطق أو الأرقام.
